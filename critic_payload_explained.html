🎨 Critic에게 보내는 값 — 유치원생도 아는 설명서

Strategist(전략가)가 판단을 끝내면, 그 결과를 한 상자(JSON)에 담아 다음 친구 Critic(반박이)에게 줘요. 이 상자 안 칸(필드) 하나하나가 무슨 뜻이고, 어디서 왔고, 왜 이렇게 정했는지를 아주 쉽게 풀어놨어요.

⭐ 내가 쓰는 상자 1개 = 아래 「Critic에 보내는 값」!

tb_strategist_signals 내 판단 결과 · 내 소유(내가 씀)

⭐ 내가 판단한 결과를 저장하는 상자. 이 상자 한 줄이 바로 Critic에게 보내는 값이에요! 그래서 아래에서 이 상자의 칸들을 하나씩 설명해요 👇

관계: 읽는 상자 5개는 ticker(종목)+cycle_id(회의번호)로 "같은 종목·같은 회의" 신호를 찾아 읽어요. 판단 후 이 상자에 1줄 저장(append).

📊 한눈 정리 — "저장하는 값 = Critic에 주는 값" (표)

내가 저장하는 값과 Critic에 보내는 값은 같은 것이에요! (tb_strategist_signals 한 행) — 따로 두 개 만드는 게 아니라, 판단 결과 1줄을 저장하고 + 매수면 그 줄을 Critic에 넘겨요. 읽는 값(공시·뉴스 등)은 입력이라 Critic에 안 가요.

표 A — 테이블 수준 (내 ERD)

→ 읽는 5개 = 입력(Critic 안 감) · 쓰는 1개 = 저장 = Critic에 보내는 값 (같은 것!)

📦 내 결과 상자(tb_strategist_signals) 속 칸 하나하나 (자세히)

ticker📥 입력

🏷️ 종목 이름표. "엔비디아"는 NVDA처럼 대문자 약어예요.

어디서: 앞단 스크리너가 고른 종목. 왜 이 기준: GPT가 이름을 잘못 쓸까 봐(NVDA→NVIDA) 코드가 강제로 확정해요.

"ticker": "NVDA"
cycle_id⚙️ 시스템

🔢 이번 판단의 묶음 번호표. "오늘 몇 번째 회의냐"를 나타내요.

어디서: 오케스트레이터(전체 지휘자)가 줘요. 왜: 같은 사이클의 공시·뉴스·기술 신호를 한 열쇠로 묶어 찾으려고요.

"cycle_id": 123
inv_type⚙️ 설정

🎯 투자 성향. 우리는 "공격형(aggressive)" 하나만 써요.

어디서: 설정값. 왜: 1차는 단순하게 공격형 한 개 (기회면 빨리 움직임).

"inv_type": "aggressive"
side🤖 GPT

🚦 살까? 가만히 있을까? buy(사자) / hold(가만히) 중 하나.

어디서: GPT가 정하고, 코드가 안전장치로 검사(확신 낮으면 hold로 강등). 왜: 1차는 매도(sell)는 안 하고 사거나·가만히만. 파는 건 나중(2차).

"side": "buy"
conviction🤖 GPT 🧮 코드보정

💪 얼마나 자신 있어? 0~1 사이 점수(소수점 3자리). 0.850이면 "꽤 확신!"

어디서: GPT가 매기고, 코드가 교차확인(+)·시장위험(−)으로 살짝 보정. 왜 0~1: 다른 점수(호재도·중요도)랑 단위를 통일하려고. 소수 3자리는 미세하게 순위를 매기려고요.

"conviction": 0.850
signal_consensus🧮 코드

🙋 "사자"에 찬성한 친구 수. 공시·뉴스·기술 3명 중 몇 명이 찬성했나(0~3).

어디서: 코드가 표를 세요. 왜 3명: 매크로(시장 날씨)는 모든 종목에 똑같아서 표에서 뺐어요(대신 감점으로만).

"signal_consensus": 3
summary🤖 GPT

📝 한 줄 요약. "왜 이 결정?"을 짧게.

어디서: GPT가 써요. 왜: Critic이 빠르게 훑어보라고.

"summary": "3개 신호가 매수 방향"
bull_case🤖 GPT

🐂 살 만한 좋은 이유. "이래서 좋아!"

어디서: GPT. 왜: Critic을 설득하려면 근거가 있어야죠.

"bull_case": "실적 서프라이즈 + 데이터센터 매출 급증"
key_risk🤖 GPT

⚠️ 제일 걱정되는 것 하나. "근데 이게 위험해."

어디서: GPT. 왜: 위험을 스스로 인정하면 Critic이 "얘 솔직하네" 하고 더 믿어줘요.

"key_risk": "기대가 너무 높아 실적 부진 시 급락 위험"
risk_rebuttal🤖 GPT

🛡️ 그 걱정에 대한 대답. "그래도 괜찮아, 왜냐면…"

어디서: GPT(+코드 태그). 왜: 위험을 미리 방어하면 Critic 반박을 이기기 쉬워요.

"risk_rebuttal": "최근 가이던스와 기술 상승세가 뒷받침"
counter_scenarios🤖 GPT

🎯 틀렸을 때 대응책. "실패하면 이렇게 할게."

어디서: GPT. 왜: 대응책까지 있으면 준비된 판단으로 보여요.

"counter_scenarios": ["실적 부진 시 -10% 손절"]
evidence🧮 코드

🔍 판단 근거가 된 숫자 모음. "왜 그렇게 정했는지 증거."

어디서: 코드 전처리 결과. 왜: 말이 아니라 숫자로 투명하게 보여주려고.

"evidence": { "buy_votes": 3, "cross_confirm_bonus": 0.02, "macro_penalty": 0, "hard_block": 0 }
sizing_hint🧮 코드

💡 "얼마나 살까?" 힌트. 근데 진짜 결정은 PM 형이 해요.

어디서: 코드가 확신도로 계산. 왜: Strategist는 순위·제안만, 실제 금액은 잔고 아는 PM 몫이라서요.

"sizing_hint": { "suggested_weight": 0.03, "conviction": 0.85, "max_weight_cap": 0.05, "stop_loss_hint": -0.05, "note": "최종 크기는 PM이 결정" }
current_price · day_high · day_low⚡ 시세

📸 판단하는 그 순간의 주식 가격 사진. 지금값·오늘최고·오늘최저.

어디서: 알파카에서 실시간(무료). 왜: 시세는 몇 초만 지나도 낡아서, 판단 순간 값을 이 칸에 박아둬요.

"current_price": 196.97, "day_high": 198.39, "day_low": 191.15
volume · turnover · high_52w · low_52w⚡ 시세

📊 거래량·거래대금·52주 최고/최저. 얼마나 사고팔렸나, 1년 중 어디쯤인가.

어디서: 알파카 실시간. 왜: 나중에 회고할 때 "그때 얼마였지?"를 정확히 남기려고.

"volume": 4767383, "high_52w": 140, "low_52w": 88
created_at⚙️ 시스템

🕒 이 판단을 저장한 시각. "언제 정했나" 도장.

어디서: 코드가 저장할 때 자동. 왜: 순서·시간을 기록해 나중에 되돌아보려고.

"created_at": "2026-07-08T11:30:00Z"

🔑 cycle_id + ticker 이 2개(FK)만 저장하면, 공시·뉴스·기술·매크로·과거 5개 원본을 다 찾아요. 원본은 복사 안 해요.

🧠 딱 3줄 정리

1. GPT가 만든 것 = 판단(side)·확신(conviction)·글(summary·bull_case·key_risk·rebuttal·대응책) 🤖

2. 코드가 만든 것 = 표 개수(consensus)·증거(evidence)·크기힌트(sizing) 🧮

3. 가져온 것 = 종목(ticker) 📥 · 실시간 시세(가격·거래량) ⚡ · 번호·시각 ⚙️
