Policy

판단 문턱값 — 안정형 vs 공격형

안정형은 "확실할 때만", 공격형은 "기회면 움직임". 모든 문턱값이 이원화되어 config.py의 POLICY에 들어감.

🛡️ 안정형 — 확실할 때만
  • 공시 sentiment / importance0.70 / 0.40
  • 뉴스 sentiment / grade / confirmed0.70 / 0.70 / 0.60
  • 뉴스 교차확인필수 ✓
  • 기술 ml_prob0.55
  • 필요 합의 수 (4개 중)3개 이상
  • 매크로 거부권 (risk-off &)risk_score ≥ 6
  • 최소 확신 (min_conviction)8.0
🔥 공격형 — 기회면 움직임
  • 공시 sentiment / importance0.60 / 0.25
  • 뉴스 sentiment / grade / confirmed0.60 / 0.50 / 0.40
  • 뉴스 교차확인선택
  • 기술 ml_prob0.50
  • 필요 합의 수 (4개 중)2개 이상
  • 매크로 거부권 (risk-off &)risk_score ≥ 8
  • 최소 확신 (min_conviction)5.0

Critic에게 보내는 것 — 설득 payload

내 판단 다음은 Risk Critic(반박 담당). 내 매수 결정에 일부러 반박해서 못 버티면 기각시킴. 그래서 "매수 8.5"만 던지면 안 되고 위험 인정 + 선제 반박 + 대응책 세트로 설득해야 통과됨.

{
  // ─── 기본 결정 (숫자) ───
  "ticker": "NVDA",
  "inv_type": "안정형",
  "side": "매수",
  "conviction": 8.5,
  "signal_consensus": 4,          // 4개 다 매수 → 강한 근거
  "cycle_id": 123,               // 원본 신호 찾는 열쇠

  // ─── 설득용 요약 (LLM 텍스트) ───
  "summary": "4개 신호 전원 매수 방향. 공시(인수)·뉴스(실적,교차확인)·기술(반등)·매크로(우호) 일치.",
  "bull_case": "① 대형 인수 공식 발표(프리미엄 30%) — 법적 확정 사실. ② 실적 서프라이즈를 Reuters·Bloomberg 교차확인. ③ RSI 34 과매도 반등권. ④ 시장 risk-on, VIX 16 안정.",

  // ─── 반박 대비 (Critic이 칠 만한 것 선제 방어) ───
  "key_risk": "인수 프리미엄 30% 과다 지불 → 단기 조정 가능",        // 위험 인정 (신뢰↑)
  "risk_rebuttal": "importance 0.8로 시장이 이미 긍정 반영 중, hard_block 없음, 안정형 기준(8.0)도 충족(8.5)",
  "counter_scenarios": [                                          // 틀렸을 때 대응책
    "인수 무산 시 → 손절 라인 -5%",
    "시장 급변(risk-off 전환) 시 → 포지션 축소"
  ],

  // ─── 신뢰 근거 (숫자 증거) ───
  "evidence": {
    "cross_source_confirmed": 1,   // 교차확인됨 → 루머 아님
    "confirmed_score": 0.67,       // 확정 비율
    "hard_block": 0,               // 차단 없음
    "macro_regime": "risk-on"
  },

  // ─── PM(Portfolio Manager)용 크기 힌트 — 제안일 뿐, 최종은 PM ───
  "sizing_hint": {
    "suggested_weight": 0.03,      // 제안 비중 3% (PM이 참고만)
    "conviction": 8.5,             // 이 확신에 근거
    "max_weight_cap": 0.05,        // 안정형 상한 5% 가이드
    "stop_loss_hint": -0.05,       // 손절 제안 -5%
    "note": "최종 크기는 PM이 결정. 이건 제안값."
  },

  "persona_notes": null            // MVP 2에서 투자 레전드 관점 추가 예정
}

payload 구성 요소 — 왜 넣나

항목역할왜 넣나
side·conviction·consensus결정 숫자기본 판단
summary한 줄 요약Critic이 빠르게 파악
bull_case매수 근거 상세설득의 핵심
key_risk위험 인정위험 알고도 판단했음을 보여줌 (신뢰↑)
risk_rebuttal위험 반박Critic 공격 선제 방어
counter_scenarios대응 시나리오"틀리면 이렇게 대응" (안전장치)
evidence숫자 증거주장이 아닌 데이터로 뒷받침
sizing_hint크기 제안PM이 참고할 제안값 (최종 결정은 PM)

원본 4개 JSON도 넘기나? — 아니, 요약만

✅ 넘기는 것

내 결정(side·conviction·consensus) · 요약·설득(summary·bull_case) · 핵심 근거 숫자(evidence) · 위험·반박·대응 · cycle_id + ticker (원본 찾는 열쇠)

❌ 안 넘기는 것

공시·뉴스·기술·매크로 원본 JSON 전체 — 이미 DB에 있으니 토큰 낭비. Critic이 의심되면 cycle_id + ticker로 직접 조회해서 검증.
